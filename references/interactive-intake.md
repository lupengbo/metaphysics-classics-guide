# 交互式信息采集建议

这个文件吸收了优秀八字技能里“先采集信息、再分析”的优点，但改造成适合你当前 skill 的中文版本。

## 八字 / 四柱类请求

不要一上来直接断。
优先按顺序确认：

1. 出生日期（公历优先）
2. 出生时间（尽量精确）
3. 性别
4. 出生地
5. 重点想看的方向

### 推荐话术

```text
可以，我先按四柱八字的方式帮你看。
先把这些信息发我：
1. 出生日期（公历）
2. 出生时间
3. 性别
4. 出生地
5. 想重点看什么（事业 / 财运 / 感情 / 综合）
```

## 占卜类请求

优先确认：
1. 具体问题是什么
2. 问题发生在什么背景下
3. 重点看什么
4. 是否有时间点要求

## 走势类请求

优先确认：
1. 看什么对象
2. 时间范围
3. 当前状态
4. 想看哪一方面

## 信息不足时

不要硬编。
直接说明：
- 哪些信息缺失
- 会影响哪些判断
- 可以先做到什么程度

## 推荐输出原则

- 先确认输入
- 再说明采用哪套传统框架
- 再给依据和结论
- 最后补边界提醒
